Some basic requirements for the final application.

•	The final deliverable must be executable on an Android tablet.

•	Information regarding staff members should be deleted from the Android tablets once the data has been successfully passed to the E-Quip system.

•	The process should be simply to learn, with clear instructions for issues and a first-time use.

•	The application should log and make obvious to the user any errors/issues which occur during uses in order to have a constant record of equipment signatures.

•	The application must operate over various/non-existent network signals.

•	The final deliverable should be well tested to ensure a quality reliable product is delivered.

•	The user should be able to select the form they wish to complete.

•	The application will follow any branding/design aspects in place within the organisation.

•	The signatures should have minimal touch delay to ensure a clear readable signature. 

•	The application will be safely integrated into any existing software such as; E-Quip.

•	The application will be supported on Android version 6 and up.

•	The application will not record/function with unnecessary staff data.

•	The application will be heavily documented to ensure that maintenance can be carried on by the next team to take on the project.

•	The team will make various on-site visits to gather more information, educate staff on how to use the system and the final install of the 
application.

•	The application will be fully completed and delivered by 25/04/2020.

•	The source code will be mainly developed using Java.

•	The signature information will not be easily accessible for any intruders and misusers.
